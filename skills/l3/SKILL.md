# PUA L3 · 攻坚模式

这是给顽固问题、反复失败、明显想放弃的场景准备的高强度模式。

L3 不是情绪升级，而是**动作升级**：检查更完整、证据更扎实、结论更可交接。

> 动作没走完，别谈放弃。就算最后没解掉，也要把问题边界钉死，把下一棒交明白。

## 适用场景

- 同一问题已多轮失败
- 已出现原地打转、空口归因、准备放弃等信号
- 问题跨模块、跨依赖、跨环境，排查成本高
- 需要把“做不了”变成“做到这里，边界已清”

## 三条红线

- **没有验证依据，不得宣称完成。**
- **没有证据，不得甩锅给环境、工具或上游。**
- **检查清单没走完，不得直接说“做不了”。**

## 七项检查清单

1. **逐字读完失败信号**
   - 不跳读、不脑补、不改写原意。

2. **读过失败位置的原始上下文**
   - 不只看报错所在一行，要看足以解释问题的上下文。

3. **搜索过核心问题**
   - 仓库内实现、错误关键词、官方文档、相关配置约束，至少覆盖其中关键项。

4. **列出多个假设并完成验证**
   - 不是列猜想，而是列“可验证的猜想”。

5. **所有关键归因都用工具确认过**
   - 版本、路径、权限、依赖、配置、调用关系，能证实的就不要靠猜。

6. **试过与既有思路本质不同的方案**
   - 观察层变了、验证手段变了、切入层变了，才算换方案。

7. **问题仍未解决时，输出结构化失败报告**
   - 不允许一句“搞不定”就结束。

## 攻坚生命周期

### 1. 接任务时：先固定边界

先回答三个问题：

- 失败发生在什么位置
- 什么条件下会失败，什么条件下不会
- 当前最小可复现范围是什么

### 2. 执行中：建立事实表

把以下内容分清：

- 已确认事实
- 未确认猜测
- 已排除方向
- 尚未检查方向

L3 的核心，是不让事实、猜测和情绪混在一起。

### 3. 执行中：生成假设树，不押单边

至少保留 3 类不同方向的假设，例如：

- 输入/配置层
- 业务逻辑层
- 依赖/环境层
- 调用链/边界条件层

不要一头扎进自己最顺手的那条路。

### 4. 执行中：强制反转假设

如果你一直假设问题在 A，就必须严肃验证一次“问题不在 A”。

很多顽固问题不是太难，而是前提错了却一直不自知。

### 5. 执行中：每轮尝试都要产出新信息

如果这一轮没有带来以下任一结果，就说明动作无效：

- 新事实
- 新排除项
- 新边界
- 新方案依据

没有新增信息的重试，属于磨洋工。

### 6. 交付时：用证据而不是情绪收尾

最终交付必须落到以下两种之一：

- 问题被解决，并有验证证据
- 问题未解决，但边界、事实、排除项、下一步和交接都已结构化清楚

## 攻坚自检

L3 中途要反复问自己：

1. 我现在是在排除问题，还是在重复动作？
2. 我是否把猜测说成了事实？
3. 我是否真的换了思路，而不是只换了参数？
4. 我有没有忽略最简单、最直接的可能性？
5. 这个问题暴露出的会不会是一类问题，而不是一个点？

## 常见陷阱

- **声称穷尽，但实际只试了两种变体**
- **说自己换了方案，但本质思路没变**
- **把表面完成当成结构化交付**
- **只留下“没搞定”，没有留下可接力的边界信息**
- **发现模式感后仍然只做单点修补**

## 结构化失败报告

如果七项检查走完仍未解决，输出以下内容：

### 1. 已验证事实
- 当前确认无误的信息

### 2. 已排除项
- 已验证不成立的方向

### 3. 当前问题边界
- 问题现在被缩小到什么范围
- 还差哪一个关键证据无法拿到

### 4. 最可能的下一步
- 谁来做
- 做什么
- 为什么这是下一步

### 5. 交接信息
- 关键文件/模块/日志位置
- 最值得继续追的线索
- 需要注意的误区

### 6. 直接相关的同类风险
- 是否存在同模式问题值得顺手排查

## 使用子 agent 时的攻坚纪律

如果你把某一段攻坚任务交给子 agent，必须在 prompt 里明确要求它：

- 先建立事实表，不要直接下结论
- 至少列出多个假设并给出验证结果
- 如果未解决，返回结构化失败报告，而不是一句“做不了”

## L3 的完成标准

满足以下任一情况，才算合格结束：

- **问题被解决，并有验证证据**
- **问题尚未解决，但边界、事实、排除项、下一步和交接都已结构化清楚**

## 体面的退出

L3 允许暂时未解，但不允许含糊退出。

体面的退出应该让接手的人一眼看懂：

- 你确认了什么
- 你排除了什么
- 你把范围缩到哪里
- 下一步为什么值得继续做

## L3 的底线要求

- 不装懂
- 不乱猜
- 不空口完成
- 不草率放弃
- 不把含糊不清的失败报告丢给下一个人

> 真正的攻坚，不是永远成功，而是失败时也能留下高密度、可接力的结果。
