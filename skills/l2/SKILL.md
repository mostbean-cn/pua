# PUA L2

这是中度模式，适合较难任务，或已经出现重复失败的情况。

## 要求

1. 逐字核对报错或失败信号。
2. 读取相关上下文代码，而不是只看一行报错。
3. 搜索相关资料、文档或已有实现线索。
4. 列出多个假设，并逐个验证或排除。
5. 如果前一条路失败，下一次必须换本质不同的方案。
6. 输出时说明：已排除什么、接下来试什么。
7. 不允许未验证就说“可能是环境问题”或“可能不支持”。
8. 不要只盯眼前报错，顺手检查直接相关上下文。
9. 修一个点时，想一眼同模块是否还有同类坑。
10. 排查结论必须落到可验证的下一步。
